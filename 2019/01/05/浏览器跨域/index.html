浏览器跨域

发表于 2019-01-05 | 分类于 http | 浏览 次

跨域限制主要是为了安全考虑，不同源的资源无法共享。

同源策略

URL地址同源指的三个相同：

  1. 协议相同
  2. 域名相同
  3. 端口相同

举例来说： http://www.example.com/dir/page.html，这个网址，协议是http,域名是www.example.com,端口是80(默认端口可以省略)，它的同源情况如下:

  • http://www.example.com/dir2/other.html 同源
  • https://www.example.com/dir2/other.html 不同源(协议不同)
  • http://www.examples.com/dir2/other.html 不同源(域名不同)
  • http://www.example.com:8080/dir2/other.html 不同源(端口不同)

AJAX跨域

同源政策规定，ajax请求只能发给同源的网址，否则就会报错。

有三种方法规避这个限制：

  • jsonp
  • websocket
  • cors

jsonp

JSONP是服务器与客户端跨源通信的常用方法。最大特点就是简单适用，老式浏览器都能完美支持。
它的基本思想就是在网页上添加<script>标签，向服务器请求json数据，这种方法利用一些标签的src不受同源策略影响（比如img的src也是如此），服务器收到请求以后，将数据放到指定的回调函数内传回来。

cors

cors是一个W3C标准，全称是”跨域资源共享”（Cross-origin resource sharing）。
cors需要浏览器和服务器同时支持，所有主流浏览器均支持此功能，ie浏览器不能低于ie10。

简单请求与非简单请求

浏览器将cors请求分为两类：简单请求与非简单请求。
只要同时满足以下两大条件，就属于简单请求。

  • 请求方法是以下三种之一:
    1. HEAD
    2. GET
    3. POST
  • HTTP的请求头信息不得超过以下几种字段:
    1. Accept
    2. Accept-Langurge
    3. Content-Langurge
    4. Last-Event-ID
    5. Content-Type: 只限于三种值application/x-www-form-urlencoded、multipart/form-data、text/plain
